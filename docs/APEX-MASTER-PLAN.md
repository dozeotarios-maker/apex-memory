## 2. AUTHORITATIVE DECISIONS (the locked stack)
- **Engine =** an **OpenCode** session running **headless + INVISIBLE** (the operator NEVER sees it; all interaction is the UI — see §13), driving **DeepSeek V4 Pro** (heavy reasoning) **+ Flash** (fast/cheap). Per-agent model assignment is **hot-swappable via config/API keys**. (Kimi was an earlier option; DeepSeek chosen.) **§19: the model layer is provider-agnostic — ANY OpenCode-supported LLM (75+ providers incl. local Ollama/LM Studio), per-role, UI-toggleable; DeepSeek V4 Pro/Flash = the DEFAULTS, not lock-in.**
- **TOPOLOGY (FINAL, latest pivot — supersedes earlier split/MCP):** **Everything in ONE Parrot OS VM** — OpenCode + the whole agent team + the tools, all co-located. **NO MCP.** OpenCode drives tools via its **native bash/shell tool**. Reasons: MCP only mattered for crossing a host↔guest boundary / multi-client; co-located, native tools suffice → far simpler, fastest v1.
  - **Blast-radius tradeoff (accepted with mitigations):** one trust domain — a VM compromise exposes DeepSeek keys + engagement data + audit. **Required mitigations:** (1) rails as **OpenCode permission rules** (`allow`/`ask`/`deny` by command pattern) + wrappers + an **egress proxy**; (2) **state/audit/knowledge on a mounted volume** (or stream audit out) so VM revert doesn't nuke the brain → keeps disposability; (3) **scoped, budget-capped, rotatable** DeepSeek keys + encrypted state volume + hardened, snapshotted VM.
  - **Reversible:** can re-introduce split + MCP in v2 if harder isolation is needed. Not a one-way door.
- **Attack-box OS = Parrot OS** (Debian-based, tools preloaded).
- **Autonomy = runtime TOGGLE: approve-gated ⇄ autonomous-all-the-way** (default approve-gated; switchable mid-engagement from the UI). *Approve-gated:* recon/low-risk runs freely, **pause for go/no-go** before irreversible/high-impact (exploit launch, lateral movement, exfil, destructive). *Autonomous:* no per-action approval. **Hard rails stay on in BOTH modes** — kill switch (immediate; a rail, not a prompt), scope/egress, no-destructive. The toggle controls ONLY per-action human approval, never the safety rails.
- **Loop control = harness-enforced + real sandbox verify** (not pure LLM self-loop).
- **Stores:** **SQLite (WAL)** for engagement state / findings / loot / audit / entity-graph (as node+edge tables); **Qdrant** (container) for RAG vectors (metadata-filtered); **filesystem** for evidence blobs. Local embedding model (e.g. `bge-base`) — offline, free per query.
- **Build/deploy:** dev on Windows, deploy on Linux. Cross-platform discipline: `pathlib`, LF, case-sensitive names, **NO machine-bound secrets** (old config derived its Fernet key from hostname+MAC → breaks portability; use env/passphrase, 12-factor). Containerize where it helps; the attack environment is the Parrot VM.
- **Tools = OSS/CLI-first.** Skip GUI/licensed tools the agent can't drive (Burp, IDA, Cobalt Strike) — humans use those; the agent uses CLI equivalents (nuclei/sqlmap/ffuf, Ghidra/radare2, Sliver/Havoc).

## 3. CORE ARCHITECTURE — "Rails + Intelligence"
LLM intelligence operates **freely inside deterministic rails (plain code)** that enforce safety/quality. The LLM is never trusted to police itself on what matters. This is also the **prompt-injection defense**: a manipulated LLM still cannot exceed scope/OPSEC/approval — the gates don't care what it was tricked into "wanting."

**The model you talk to (Conductor) PLANS and DELEGATES execution** — planning stays with the smart model that holds the conversation (intent→strategy fidelity); noisy execution is pushed into disposable subagent contexts (context hygiene = also the #1 token saver).

**Broadcast:** the Conductor owns a **shared engagement-directive block** injected into every agent's prompt. "Inject a prompt to the whole team" = update that block → all agents pick it up on their next step. (Kill switch is the exception — immediate, deterministic.)

## 6. THE RAILS (deterministic; re-homed in-VM as OpenCode permissions/hooks/wrappers + egress proxy)
- **RoE artifact** — structured authorized scope (IP/CIDR/domain), time windows, forbidden actions, data-handling. The source of truth the scope gate checks.
- **Scope gate** — **egress proxy** validates every destination vs the RoE allowlist; blocks cloud-metadata (`169.254.0.0/16`) + private ranges unless in scope. Packet-level, not just prompt-level.
- **OPSEC gate** — pre-action detection-risk score + noise budget; block / downgrade / queue-for-approval.
- **Approval gate** — OpenCode `"ask"` permission on risky tools (native).
- **Kill switch** — immediate hard stop.
- **Pivot Controller** (§4). **Deconfliction** — locks so parallel agents don't collide. **Cost/Concurrency governor** — token budget, max-concurrent, throttle. **Audit log** — append-only, streamed/mounted external (survives VM compromise/revert).
- **Counter-Detection & Ghost:** radar (access revoked / lockout / honeytoken tripped / foothold goes dark / C2 anomaly / EDR appears / sandbox-deception fingerprint) → response playbook (go quiet / pivot infra / **burn-and-abandon** burned foothold / preserve **compartmentalized** others / escalate) → **footprint ledger + anti-forensics** (track every artifact, clean carefully, **RoE-bounded, never destructive, OWN host audit always preserved**) → compartmentalization (isolated C2 channels/keys so one burn doesn't cascade). Honest: can't perfectly know you're detected → default **low-and-slow + ready-to-burn**.
- **Injection defense:** all external content (tool output, web, banners, repos) = **DATA, not instructions**; rails are the backstop.
- **No-destructive guardrail:** hard block DoS/data-destruction outside authorized impact.
- **Human-fallback rail (escalate, don't fail) — the operator is the last resort, not the orchestra:** when a task exceeds the agents' reach — a capability gap the §7 trifecta can't close · a human-only action (physical/SE) · a hard blocker (repeated failure / missing access / dead-end) · genuine ambiguity or an out-of-RoE situation — the **Supervisor escalates to the human** (deterministic, NOT LLM discretion — an agent can't be trusted to "decide to give up honestly"; the loop-controller + capability checks detect the stuck/gap condition and route to you). Surfaces a clear **"needs you"** escalation in the UI (extends the approval queue), pauses that objective thread with context preserved, and resumes on your input (provide the capability · do the human-led action · answer · re-scope · redirect). **Stays ON in BOTH autonomy modes** — autonomous suppresses per-action *approval*, never a genuine *can't-proceed*. Honest-failure principle: faked or low-confidence work is worse than asking.

## 15. CONTEXT-WINDOW MANAGEMENT & ENGAGEMENT MEMORY (2026-05-28) — AUTHORITATIVE
Extends §3 (Conductor delegates / context hygiene), §10 (token efficiency), §13 (resilience/observability/agent-identity). Resolves "what happens when context windows degrade mid-engagement." A real engagement runs hours→days; the Conductor's window both **overflows** (hard limit) and **rots** (quality degrades well before the limit). → This is a **v1 pillar, not a v2 add-on** (it de-risks v1: a long op falls over without it).

**THE REFRAME (keystone):** *durable engagement memory is the brain; the LLM context window is disposable scratch* — a freshly-assembled, bounded **projection** of that memory, scoped to the current task. The transcript is NOT the source of truth (SQLite + the vault are). So any window can be shrunk/reset and **rehydrated** losslessly. **Safety corollary:** the rails do NOT live in any context window — scope/kill/approval/cost are Supervisor code — so a blown/rotted context can make the operator dumber or repetitive but **NEVER less safe**. Context management is a capability/quality/cost concern, cleanly severed from safety.

**DECISION — Path B: build a THIN vault layer over our OWN stack; do NOT adopt a memory framework.** Researched the 2025–26 agent-memory landscape (Letta/MemGPT, Mem0, LangMem, Graphiti/Zep, Cognee, A-MEM, Basic Memory, sqlite-memory; full scorecards in session research transcripts). **Verdict: every "agentic memory" framework is REFERENCE-ONLY for APEX** — their headline value is *an LLM editing/extracting memory on the write path* = non-deterministic (anti-"consistent"), token-spending (anti-"lean"), and a violation of the Supervisor-as-pager rule; most also force a redundant heavyweight store (Letta→Postgres+pgvector, Graphiti→Neo4j, Memary→Neo4j) and are DB-opaque (no Obsidian). The closest ADOPT candidate, **Basic Memory** (it *is* "Obsidian+Claude": markdown-as-truth + SQLite index + local bge + **deterministic non-LLM writes** + JSON CLI, no MCP), was **rejected as a dependency** on three grounds: (1) **AGPL-3.0** network-copyleft (APEX's web UI = network use; modifying a core dep risks source-disclosure — unacceptable for client-facing/proprietary tradecraft); (2) **files-as-truth + weak multi-writer concurrency** vs APEX's multi-agent parallel recon (its weakest axis = our normal case); (3) a **second SQLite + second vector store** (sqlite-vec) to keep coherent with our `engagement.db` + Qdrant. **Mem0 is OUT for the live path** (LLM-on-write; DB-opaque; graph paywalled + OpenAI-hardcoded; "database is locked" under concurrent agents) — bookmarked only for optional *post-engagement* auto-extraction. **Why B is better, not heavier:** the vault rides on the SQLite-WAL + Qdrant + embeddings we build for the entity graph/findings/audit ANYWAY → marginal cost ~2–3 days of glue, not a new subsystem; we reuse proven **blueprints** (Basic Memory's entity/observation/relation + `memory://` paging; Graphiti's bi-temporal schema; sqlite-memory's hash-aware indexing) without the dependency/license/coupling; and we get *stronger* consistency than any file-first tool. Over the tool's life B is lower-TCO and lower-risk. **Small permissive libs reused:** python-frontmatter · markdown-it-py (wikilink/tag→edges) · SQLite FTS5 · bge local embeddings · git/GitPython · watchfiles (optional). *(A wins only if APEX were personal/internal [AGPL moot], single-agent [concurrency moot], or a throwaway spike — none hold.)*

**CONSISTENCY KEYSTONE (the "absolute best consistent memory" answer) — adopt Graphiti's *pattern*, not its engine:** bi-temporal facts in SQLite-WAL — every entity/edge/finding carries `valid_from / valid_to / observed_at / invalidated_at`; an **invalidate-don't-delete** rule (port open→patched, cred valid→rotated, host up→down never lose history); content-hash dedup; per-entity/per-note locks. The **Supervisor is the single writer** (serialize through one queue; read-modify-write under lock) → genuinely transactional, full audit history, contradiction handling, renders straight to Obsidian markdown. **Zero new infra.**

**MEMORY LAYERS (durable vs working):** ① **Durable state = source of truth** (SQLite-WAL, already planned): entity graph · findings · objective tree · loot · audit · timeline + the bi-temporal columns. *Queried, not "remembered."* ② **Episodic memory = experience** (decision journal in SQLite + Qdrant **engagement-scoped** collection, separate from the §8 knowledge collection): decisions+why · hypotheses→outcomes · pivots · lessons → powers `recall()` + anti-repetition. ③ **Working context = scratch** (the LLM window): role prompt (cached prefix) + situation brief + recent turns only. ④ **Knowledge = external** (§8, unchanged). Plus a small agent-editable **`WORKING_MEMORY.md` core block** (current focus · live hypotheses · next intended step) always in the brief, surviving every reset (MemGPT core-memory idea).

**THE VAULT (state-primary, vault-as-VIEW — the earlier "Option B"):** SQLite is truth; the Supervisor **renders a git-backed Obsidian-style vault** (one note per entity/finding with `[[backlinks]]` + YAML frontmatter, an `INDEX.md` MOC, per-target folders under `engagements/<id>/targets/<t>/`) as the human-readable graph. Agents **read** the vault (page in by link/search), **write facts** via structured tools (validated → SQLite → re-render), **write narrative** (decisions/handoffs) as free markdown notes. **Git-back the vault** → every memory mutation = an auditable, revertable diff (free memory audit log; *eviction ≠ deletion* is literal). APEX surfaces memory through its OWN 3-tab UI (§13), not Obsidian — Obsidian-compat is a "human can peek" side-benefit.

**CONTEXT LIFECYCLE — Supervisor-owned compaction (deterministic, like kill/cost):** MemGPT-style paging, but the **Supervisor is the pager** (never trust the LLM to manage its own memory) and **structured state is primary** (not a vector blob). **Triggers:** an ABSOLUTE sharp-zone token target (tens of thousands, NOT a % of the 1M window — §18 refinement) (compact at next OODA boundary — target the "sharp zone": small context reasons *better*, so this is a quality lever, not just overflow) · hard ~85% (force) · task/phase boundary (cleanest). **Compaction = persist-first → assemble → reset:** (1) *persist-first* — everything material is already in durable state (parse-before-feed + event stream did it), so compaction mostly *discards redundancy*; (2) *deterministic situation brief* (Supervisor code): current objective+parent chain · active target/segment top-N entities · open leads/in-flight hypotheses · recent decisions · OPSEC posture · **active broadcast directive (must survive reset)** · pending approvals · already-tried-here (anti-repetition); (3) one bounded LLM **"shift-change" note** → journal (supplement, not truth); (4) *reset* to `[immutable cached prefix: role+tools+static RoE] ++ [volatile: brief+handoff+recent turns]` — **preserves the DeepSeek prefix cache** (only the volatile tail rebuilds). **Eviction ≠ deletion:** raw pre-compaction transcript archived to `sessions/`, never destroyed. **Memory-access tools** (keep working context small by making detail *fetchable*): `query_state(...)` (graph/findings/objective/loot) · `recall(query)` (episodic semantic) · `get_evidence/finding(id)`. **Born-scoped-die workers:** only the Conductor has long continuity (the main compaction target); Operator/Analyst spawn per task → persist → tear down → never accumulate.

**MAIN-AGENT MEMORY ACCESS (INVARIANT — "dive into everything, pay for almost nothing"):** the Conductor must reach **ALL** engagement memory on demand while keeping its window lean — *completeness ≠ residence in context*. Guaranteed by: (1) a compact **memory map/manifest** in every brief (live inventory — N hosts · M creds · K findings · objective-tree status + pointers — so it always knows what EXISTS without holding it); (2) **completeness via deterministic structured query (`query_state`) + the navigable vault INDEX/backlinks**, NOT just probabilistic semantic `recall()` (vector search alone can miss — the relational query + MOC graph is the exhaustive guarantee that nothing is unreachable); (3) **fetch-don't-hold** — tools return compact, distilled answers, never raw dumps (parse-before-feed applies to memory too); (4) heavy/broad digs **delegated to a disposable "librarian" retrieval sub-agent** that searches and returns a small result, protecting the Conductor's context; (5) **prefix-cached** tool/system defs so repeat queries are near-free. Net: the Conductor behaves as if it has perfect total recall, while its context holds only the map + the current working set.

**CROSS-SESSION RESUME (the payoff):** because context is a reconstructable projection, **crash / pause / a 3-day engagement are the SAME mechanism** — N compaction/rehydration cycles over one durable memory. Supervisor reloads the engagement, re-assembles the brief, resumes the state machine at the last checkpoint (fully realizes §13 RESILIENCE — same code path as compaction).

**FAILURE MODES handled:** mid-action compaction (only at safe boundaries; **live handles — session IDs/tunnels/C2 — MUST be durable state**, also an OPSEC/cleanup requirement) · lossy summary (persist-first + structured brief + archived transcript; only scratch reasoning at risk, covered by the handoff note) · memory poisoning via recall (entries carry provenance+trust per §8; recalled memory = quoted DATA, never instructions; rails gate regardless) · SQLite↔Qdrant desync (SQLite = truth, Qdrant = rebuildable index, idempotent re-embed) · huge engagement (brief = active focus + counts/pointers; relevance-ranked; deterministically size-bounded).

**OpenCode-API sub-requirements (sharpens §13 open #1):** the integration must expose (a) **per-session token usage** (to budget), (b) **session seed/restart** (a "context epoch" = a fresh headless session seeded with our brief, sharing durable memory), (c) ability to **DISABLE OpenCode's native compaction** (we own compaction; letting it lossily summarize behind our back breaks determinism + invisibility). If unavailable → fallback CLI/hooks, each epoch = a new headless invocation.

## 16. DELIBERATION FABRIC — the Co-Scientist, engineered for speed (2026-05-28) — AUTHORITATIVE
Iterates §5 (assurance model) + §4 (Co-Scientist mapping). **DECISION: the standing-orchestra core + the Co-Scientist-derived assurance engine + per-agent system prompts are all v1** (moved up from v2), engineered **cheap-by-default**. Backed by a 4-lane 2024–26 research sweep (multi-agent debate · efficient ranking/LLM-judge · test-time-compute + self-correction limits · Co-Scientist + autonomous-pentest prior art; full cited briefs in session transcripts).

**THE THESIS (APEX's economic inversion):** Google's Co-Scientist spends unbounded test-time compute on Elo tournaments + evolution because **open biomedical hypotheses have no cheap oracle**. **APEX has one: the target/sandbox.** "Does this exploit work / is the box rooted / is the port open / does the parse match schema" is testable in seconds. Evidence is decisive: LLM self-correction **collapses without ground truth** (Huang et al. ICLR'24: GSM8K 80.2→76.6, GPT-4 92.0→88.7 under intrinsic self-critique) but **sound external verification gives large gains** (Stechly/Kambhampati 2024); multi-agent **debate does NOT beat cheap self-consistency at matched compute** and degrades via conformity/sycophancy (multiple 2024–26 replications); adaptive compute beats uniform **~4×** (Snell 2024); **"abstraction beats model size"** (Incalmo 37/40 vs 3/40). **MASTER RULE: don't debate what you can test.** This inversion is what makes "fast + efficient without losing quality" real.

**THE FABRIC (one uniform, Supervisor-owned service every role's output flows through; Supervisor sets the tier deterministically from stakes):**
- **GATE 0 — Oracle-first (dominant path, most decisions):** testable fact + a probe/sandbox exists → **execute the test; treat as ground truth.** Never route testable facts to LLM debate/self-verify. Biggest token saver; immune to debate's echo-chamber failure.
- **T0 — Reflex (Flash, 1 pass):** routine/reversible/in-distribution (parse · pick next host · format cmd). No deliberation. Bulk of actions.
- **T1 — Sample-and-check (Flash, DISAGREEMENT-gated):** sample k≈3; agree→accept, disagree→escalate. Gate on **empirical dispersion, NOT self-reported confidence** (LLM confidence miscalibrated, worst in adversarial/OOD = pentest). For generation the "check" is the oracle, not a vote. Cap k≈5 (diminishing returns).
- **T2 — Critique (Pro, one-shot, independent):** survived-T1-disagreement or medium-stake. *Different*-role critic (Reviewer + OPSEC lens), candidates **anonymized**. One strong pass — NOT multi-round debate (low marginal value + herding). FrugalGPT-class cascade jump.
- **T3 — Deliberate + rank + gate (Pro-heavy, parallel; full machinery — RARE):** ONLY irreversible/engagement-defining (fire exploit on prod · lateral move · exfil · scope-boundary · major pivot · OPSEC go/no-go). Pipeline: **generate N (3–5 diverse, bounded action space) → Reflection deep-verify (decompose→check each + scope/OPSEC) → rank (listwise rubric = 1 call default; swap-checked pairwise only on contested top-2; 3-model PoLL jury for highest stakes) → bounded evolve (param-mutate/simplify, ≤2 gens) → decision → human gate.** **Stake-floor OVERRIDES the confidence gate** — these always get T3 even if "confident."

**EFFICIENCY INVARIANTS ("fast without quality loss"):** (1) **oracle > critic > debate** (soundness beats cleverness — spend budget on real probes, not bigger critics); (2) **gate on empirical disagreement, not self-confidence** (survives adversarial/OOD miscalibration); (3) **difficulty-adaptive METHOD** (Snell ~4×): easy→sequential refine, hard→**go WIDE (parallel sample) not deep**; (4) **right model per stage = abstraction>size**: Flash for generate-bulk/triage/listwise/dedup/parse, Pro for hard planning/deep-verify/final-ranking-on-close-calls; (5) **bounded + forced decision** (caps: candidates 3–5 · debate ≤2 rounds · evolve ≤2 gens · hard OODA/token/wall-clock budget per objective — prior art 12–54 min, ~$15–24/run); (6) **state OUT of context** (query §15 memory/entity-graph, never stuff — the silent token killer); (7) **disable intrinsic self-correction on ungrounded reasoning** (degrades accuracy); only reflect with a real feedback channel, cap loops ~2 rounds (~75% of gains); (8) **bias controls on ALL ranking**: swap-and-average pairwise · shuffle listwise (2 shuffles, reconcile) · anonymize · cross-family/jury for high-stakes · verbosity-normalize · never self-judge own slate; (9) **meta-review amortizes** (failure patterns → prompts + §15 memory → future decisions deliberate less); (10) **cache** (prefix-cache rubrics/system via DeepSeek context caching; result-cache identical sub-evals §10).

**CO-SCIENTIST → APEX DISPOSITION:** Supervisor + async worker-pool + context-memory = **ADOPT faithfully** (our Supervisor + scheduler §10 + §15 memory; statistic-driven agent reweighting + restart-on-failure — most transferable piece) · Meta-review = **ADOPT** (cheap no-weight-update learner) · Reflection/deep-verify = **ADOPT, repurpose** (pre-exec OPSEC/scope gate + post-exec evidence validator) · Generation = **ADAPT** (bounded 3–5 candidates over finite technique/ATT&CK space, lower temp) · Ranking/Elo = **ADAPT + DE-RISK** (listwise-default, pairwise-only-contested; prefer **environment-grounded probe scores over LLM-judge**; Elo FRAGILE — 394–1,122-pt swings, circular self-judge → do NOT persist as truth; soft seed + bias controls only) · Proximity graph = **DROP graph, keep cheap dedupe** (hash/embedding over action-history = anti-repetition §4 + repetition-penalty) · Evolution = **MOSTLY DROP** (replace with bounded param-mutation-on-failure + "cheapest action that works"; novelty-search wrong objective for a bounded op).

**PENTEST PRIOR-ART folded in (PentestGPT/HPTSA/Incalmo/Pentest-R1, 2023–26):** hard planning/execution split (Conductor plans / Operator executes) · external structured world-model kept OUT of context (= §15 + entity graph; biggest measured token saver) · **environment-grounded success verification** (shell/flag/asset, consistent over N trials — never LLM-self-declared) · action-history + repetition penalty for loop control · per-objective budget + Supervisor stop/escalate-to-human gate · "abstraction beats model size."
